\begin{abstract}
Se desarrolló un programa en C que codifica y decodifica información en formato base 64. Asimismo, parte del código se realizó directamente en assembly MIPS32; en particular las funciones C: int base64_encode(int infd, int outfd) e: int base64_decode(int infd, int outfd). Más cualquiera otra función que fuera derivada como solución de éstas dos.
El objetivo del presente trabajo fue familiarizarse con las principales instrucciones del lenguaje assembly MIPS, y con el concepto de ABI (application binary interface).
\end{abstract}

\section{Introducción}
Se implementó en C un codificador/decodificador de información en base 64.
Base 64 es un sistema de numeración posicional que usa 64 símbolos como base. Es la mayor potencia de dos que puede ser representada usando únicamente los caracteres imprimibles de ASCII. Esto ha propiciado su uso para codificación de correos electrónicos, PGP y otras aplicaciones.
Como puede entenderse, se trata de un sistema simple por lo que resulta una buena elección si el objetivo final no es construir un sistema elaborado sino familiarizarse con el lenguaje assembly de la arquitectura MIPS, más ciertas especificaciones de ABI.

\subsection{Especificaciones}

El funcionamiento del algoritmo de encoding/decoding es el siguiente:

* Para codificar:
Se toma una cantidad establecida de bytes del mensaje a codificar (3 caracteres para el caso en cada ciclo), y se obtienen los seis bits siguientes necesarios para un símbolo en base64, utilizando decalajes (instrucciones MIPS sll y srl). Asi, cada word de 32 bits con sus seis bits menos significativos (parte derecha de la secuencia de bits), indica el código correspondiente en base 64, guardado en una tabla con todas las posibles secuencias de caracteres que puede tomar un texto en dicha base. Luego guarda cada símbolo correspondiente en talba -del nuevo mensaje codificado- en un archivo de salida.

* Para decodificar:
El camino es inverso a la codificación, se toma una cantidad establecida de bytes del mensaje a decodificar (4 caracteres para el caso en cada ciclo), y se obtienen los ocho bits siguientes necesarios para un símbolo en la tabla base64, mediante decalajes (instrucciones MIPS sll y srl). Asi, cada word de 32 bits con sus ocho bits menos significativos (parte derecha de la secuencia de bits), coincide con un caracter código en la tabla base64. Mediante las posiciones de éstos en la tabla recostruye los 8 bits de los caracteres originales. Luego guarda cada caracter ASCII en un archivo de salida.

\section{Compilación}
Los argumentos utilizados para la compilación son los siguientes:

\begin{description}
\item[-c] Compila el código fuente pero no corre el linker. Genera el código objeto.

\item[-o] Especifica el archivo de salida (ya sea un archivo objeto, ejecutable, ensamblado).

\item[-Wall] Activa los mensajes de warning.

Se usaron las siguientes lineas de comandos bash para la generación del ejecutable
\begin{description}
	\item[] gcc -c -ggdb -Wall tp1.c
